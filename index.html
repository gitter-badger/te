Application Programming Interfaces

A new course to be taught by Karl Stolley at IIT in Fall 2013

Suppose you want to get a list of all of the cars towed in the city of Chicago yesterday. Suppose you want also to display that list on a Google Map, without having to manually enter data points? Or perhaps you wanted to pull down all of the comments on a particular story from the New York Times for computational analysis. Or maybe you're ready to build the next killer app for iPhone or the open Web, using openly available data from government or private sources.

This class is for you.

The Application Programming Interface (API) is a broad concept for making software tools and data accessible to do exactly those things. The focus of this course will be web-based data APIs, but we will begin by looking at APIs for software libraries and frameworks, particularly Ruby and JavaScript and JavaScript projects such as jQuery and Node.js (we will use those to build projects for accessing and manipulating data APIs and, later, building our own APIs).

The course will be divided into two parts: API consumption and API production. Within the consumption part, which will be about two-thirds of the course, we will look at programming APIs (e.g., for jQuery) and web-available data APIs. The creation portion that makes up the last third of the course will focus on the creation of data APIs in JSON, and building their documentation.

Some Course Goals

  • Locating open data APIs and gaining access to restricted APIs
  • Working with API keys and authentication
  • Learning effective strategies for navigating and reading API documentation
  • Understanding common data API response and data serialization formats, especially JSON
  • Mashing up the data from one API with the data from one or more additional APIs
  • Building and documenting RESTful web APIs that others may use

Some Course Readings

  • David Copeland, Building Awesome Command-Line Applications with Ruby
  • Marijn Haverbeke, Eloquent JavaScript
