Application Programming Interfaces

A new course to be taught by Prof. Karl Stolley at IIT in Fall 2013.

What do the Chicago Transit Authority, The New York Times, Instagram, Twitter, and fans of the band Phish have in common? Each offers data APIs that provide some level of access to data generated and curated by the organization and its users. All of those services are in the company of thousands of others that provide access to data and services ranging categories from weather to music to shopping.

If you’re looking to learn how to access, archive, repackage, and/or create new mashups of open data, this class is for you. Whether your purpose is to create a dataset for analysis or to build the next killer Web or mobile app, you’ll learn how to understand and work with a wide range of protocols and response formats common to data APIs. You’ll also learn to navigate the sometimes spare documentation for APIs, and work within the constraints of a given API’s terms of service. And if you have or are looking to build a data set to share openly with the rest of the world through your own API, you’ll learn how to do that, too.

Wait, what’s an API?

The Application Programming Interface (API) is a concept for exposing functionality and data of a system or service. APIs make it possibile to access software functionality or system data within and across different platforms. The focus of this course will be web-based data APIs, but we will begin by looking at APIs for software libraries and frameworks, particularly Ruby and JavaScript and JavaScript projects such as jQuery and Node.js (we will use those to build projects for accessing and manipulating data APIs and, later, building our own APIs).

The course will be divided into two parts: API consumption and API production. Within the consumption part, which will be about two-thirds of the course, we will look first at programming APIs (e.g., for jQuery) and then more extensively at web-available data APIs. The creation portion that makes up the last third of the course will focus on the creation of RESTful data APIs in JSON, and writing documentation to describe access methods and procedures for other people.

Some Course Goals

Students in the course can expect to learn to:

  • Locate open data APIs and gain access to restricted APIs
  • Work with API keys and authentication
  • Develop effective strategies for navigating and reading API documentation
  • Understand common web-API response and data serialization formats
  • Mash up the data from one API with the data from one or more additional APIs
  • Build and document RESTful web APIs that others may use
