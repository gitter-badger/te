Application Programming Interfaces

A new course to be taught by Karl Stolley at IIT in Fall 2013.

What do the Chicago Transit Authority, The New York Times, Instagram, Twitter, Facebook, and fans of the band Phish have in common? Each offers data APIs that provide some level of access to data generated and curated by the organization and its users. All of those services are in the company of thousands of others that provide access to data and services ranging categories from weather to music to shopping.

If you're looking to learn how to access, archive, repackage, and/or create new mashups of open data, this class is for you, whether your purpose is to create a dataset for analysis or to build the next killer Web or mobile app. You'll learn how to understand and work within each data API's terms of service. And if you have or are looking to build a data set to share openly with the rest of the world through your own API, you'll learn how to do that, too.

The Application Programming Interface (API) is a broad concept for making software tools and data accessible to do exactly those things. The focus of this course will be web-based data APIs, but we will begin by looking at APIs for software libraries and frameworks, particularly Ruby and JavaScript and JavaScript projects such as jQuery and Node.js (we will use those to build projects for accessing and manipulating data APIs and, later, building our own APIs).

The course will be divided into two parts: API consumption and API production. Within the consumption part, which will be about two-thirds of the course, we will look first at programming APIs (e.g., for jQuery) and then more extensively at web-available data APIs. The creation portion that makes up the last third of the course will focus on the creation of data APIs in JSON, and writing documentation to simplify access for others.

Some Course Goals

  • Locating open data APIs and gaining access to restricted APIs
  • Working with API keys and authentication
  • Learning effective strategies for navigating and reading API documentation
  • Understanding common data API response and data serialization formats, especially JSON
  • Mashing up the data from one API with the data from one or more additional APIs
  • Building and documenting RESTful web APIs that others may use

Some Course Readings

  • David Copeland, Building Awesome Command-Line Applications with Ruby
  • Marijn Haverbeke, Eloquent JavaScript
